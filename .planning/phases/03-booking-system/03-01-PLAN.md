<task type="auto">
  <name>Task 1: Add Booking model to schema and run migration</name>
  <files>prisma/schema.prisma</files>
  <action>
Append the following model to `prisma/schema.prisma` after the existing `OAuthToken` model. Do not modify any existing model.

```prisma
model Booking {
  id              Int      @id @default(autoincrement())
  name            String
  email           String
  phone           String?
  message         String?
  slotTime        DateTime @unique  // DB-layer double-booking lock — P2002 fires on duplicate
  calendarEventId String?           // populated after Calendar event created; null if creation failed
  createdAt       DateTime @default(now())
}
```

After saving the schema, run the migration:
```bash
npx prisma migrate dev --name add_booking_model
```

This generates a migration file under `prisma/migrations/` and regenerates the Prisma client at `src/generated/prisma`. The client regeneration is required so downstream plans can import the `Booking` type.

Verify the migration applied cleanly — the command should print "Your database is now in sync with your schema." If it prompts for a name, use `add_booking_model`.
  </action>
  <verify>

  </verify>

`npx prisma migrate status` reports all migrations applied with no pending migrations. The file `prisma/schema.prisma` contains `model Booking` with `slotTime DateTime @unique`. The `src/generated/prisma` directory contains an updated client with the `Booking` type.
  </done>
